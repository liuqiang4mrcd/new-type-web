# H5 活动页设计规范

> 来源：Figma MCP 提取 + 设计师 UI 默认规范  
> 提取日期：2026-04-09

---

## 适用范围

- 移动端 H5 活动页
- 品牌宣传页
- 落地页
- 表单页
- 图文营销页
- 轻交互专题页

如果用户没有额外指定风格、平台、品牌规范或终端形态，默认按本规范进行设计。

---

## 设计目标

- 优先保证信息传达清晰。
- 优先保证移动端浏览体验。
- 优先保证模块层级明确、视觉节奏稳定。
- 优先保证设计可落地、可实现、可还原。
- 避免为了"视觉复杂"牺牲可读性、可点击性和开发成本。

---

## 一、画布 / 页面基础

- 默认按移动端 H5 设计，设计稿宽度 `750px`（@2x，对应 375px 视口）。
- 所有尺寸、字号、边距、圆角、描边、阴影偏移等设计值，均按 750 画板输出。
- 页面内容需考虑主流手机的安全展示区域，不要将关键信息贴边放置。
- 首屏需优先保证核心标题、核心卖点、主按钮在常见手机尺寸中清晰可见。

| 属性 | 值 |
|---|---|
| 设计稿宽度 | **750px**（@2x，对应 375px 视口） |
| 开发写法 | 设计稿 px 直接写入样式代码，构建时 `postcss-mobile-forever` 按 `viewportWidth: 750` 自动转为 vw |
| 页面底部留白 | 90px |
| 布局方式 | 纵向滚动，flex-col |

---

## 二、页面结构规范

H5 页面默认采用模块化组织，不建议整页作为单一大图或单一超长容器处理。

建议结构：

- 顶部视觉区
- 核心标题与卖点区
- 内容模块区
- 功能或权益说明区
- 操作引导区
- 底部补充信息区

要求：

- 页面首屏必须有明确主视觉焦点。
- 每个模块必须有清晰边界，避免模块之间粘连。
- 长页面应通过背景变化、卡片分组、留白或分隔带形成节奏。
- 模块顺序要符合用户阅读路径，不要频繁打断认知流。

---

## 三、布局规范

- 优先使用清晰的纵向信息流布局。
- 默认保持单列布局；如有并列内容，应保证对齐和节奏统一。
- 避免大量漂浮元素破坏阅读路径。
- 避免纯依赖绝对定位叠出复杂视觉，除非主视觉区确有必要。
- 同一页面中，卡片、图文块、按钮组应尽量使用统一的对齐逻辑。

---

## 四、色彩规范

### 4.1 色彩原则

- 默认采用主色、辅助色、中性色的分层方式。
- 主色用于核心 CTA、关键信息强调、品牌主视觉。
- 辅助色用于局部强化、状态区分、模块点缀。
- 中性色用于正文、说明、分隔、背景、描边。
- 避免同一页面出现过多高饱和颜色抢夺焦点。
- 对比关系要明确，尤其是按钮文案、标题和正文的可读性。
- 大面积背景色需要兼顾内容可读性，不要让背景压过信息本身。

### 4.2 字体主体色

| 用途 | 色值 |
|---|---|
| 主金色（高亮标题/金额） | `#ffe45a` |
| 次金色（标题/装饰） | `#ffd736` |
| 金色3（硬币数值） | `#ffdc5c` |
| 深红（金额标签文字） | `#910000` |
| 深色文字 | `#333333` |
| 白色文字（高亮标题/金额） | `#ffffff` |
| 红色警示/进度文字 | `#ff6a6a` |

## 五、字体规范

### 5.1 字体原则

- 默认使用简洁、稳定、可读性高的字体风格。
- 标题、正文、说明、按钮文案需有明确层级差异。
- 正文默认不小于 `10px`（CSS px，对应设计稿 10px）。
- 次级说明文字应保持可读，不使用过小字号硬塞信息。
- 标题字号应根据层级递进，避免多个层级使用过于接近的字号。
- 行高应服务于移动端阅读，避免文字过密。
- 大段文案应优先考虑分段、列表化或弱化处理，不建议长段连续堆叠。

### 5.2 字体族

| 字体 | 用途 |
|---|---|
| `Roboto Black` | 标题、金额、按钮、标签数值 |
| `Roboto Bold` | 倒计时、强调文字 |
| `Roboto Medium` | 正文、说明、副标题 |
| `Shrikhand Regular Italic` | 排行榜 TOP 标题 |

## 六、间距规范

### 6.1 间距原则

- 页面必须建立统一的间距系统，避免同页中出现随意变化的边距。
- 模块左右边距默认保持统一。
- 模块上下间距要体现层级：模块间距大于模块内间距。
- 标题与正文、正文与按钮、图片与说明之间应保持稳定节奏。
- 不要通过极小间距堆积信息，也不要通过过大留白制造空洞感。

### 6.2 间距数值（750px 设计稿）

| 间距 | 用途 |
|---|---|

| 50px | 部分区块水平 padding |
| 30px | 章节间距 / 卡片间距 / content_mid 水平 |
| 20px | 组件内 gap / tab 左右 padding |
| 14px | 标签底部 padding |
| 10px | 内容元素间距 / tag padding |

## 八、图形与图片规范

- 图片需服务于信息表达，不应只作为装饰堆积。
- Banner 图、主视觉图、插图、商品图、人物图需职责明确。
- 同一页面中的图片风格要统一，不要混用明显不同质感的素材。
- 裁图应避免截断主体、文字或关键视觉元素。
- 若使用背景图，需保证前景文字仍清晰可读。
- 装饰性元素应适量，避免过多光效、描边、悬浮贴纸导致页面杂乱。

---

## 九、卡片与分组规范

- 卡片是 H5 页面最常见的信息容器，应统一圆角、内边距、阴影和边框风格。
- 同级卡片建议保持一致尺寸逻辑和视觉语言。
- 卡片内信息层级应明确，通常包含标题、说明、主内容、补充信息、操作区。
- 不要让卡片承担过多层级嵌套，避免出现"卡片套卡片套卡片"。

---

## 十、按钮与操作规范

- 每个页面必须有明确主操作和次操作层级。
- 主按钮视觉强调度必须最高。
- 次按钮不应与主按钮争夺注意力。
- 按钮尺寸应适应移动端点击习惯。
- 按钮文案应直接表达动作，不使用模糊描述。
- 同一页面内按钮样式数量应受控，避免每个模块都是不同按钮体系。

---

## 十一、表单与输入规范

- 表单项需具备明确标签、输入区、提示信息和状态反馈。
- 必填、选填、错误、禁用、完成等状态应设计清楚。
- 输入区尺寸和间距应满足移动端点击与输入习惯。
- 表单页面应优先减少一次性暴露的信息量，降低填写压力。
- 提示文案要简洁，错误信息要能直接指导修正。

---

## 十二、图标与装饰规范

- 图标风格需统一，不混用线性、面性、拟物等差异过大的体系。
- 图标优先辅助理解，不承担过多装饰负担。
- 小图标若具备点击含义，应保证视觉识别和点击热区。
- 装饰性图形需服从内容，不应喧宾夺主。

---

## 十三、视觉层级规范

- 每个页面必须存在清晰的第一视觉焦点。
- 页面层级一般为：主标题、核心卖点、主要内容、辅助信息、弱化说明。
- 不要在同一屏内同时出现多个视觉中心。
- 强调信息应通过字号、字重、颜色、位置、留白综合体现，而不是只靠颜色。

---

## 十四、动效与状态规范

- 默认仅使用必要动效，不追求无意义的复杂动画。
- 动效应服务于状态变化、页面切换、反馈提示和视觉引导。
- 动效节奏应简洁自然，避免拖沓。
- 点击态、选中态、禁用态、加载态、完成态应有明确视觉区别。

---

## 十五、可读性与无障碍规范

- 正文与背景的对比度必须足以支持稳定阅读。
- 重要信息不能只依赖颜色表达。
- 小字、浅字、透明字要谨慎使用。
- 长列表、长文案和复杂信息区应优先提高信息扫描效率。
- 设计时应默认考虑触控操作、弱光环境和常见移动端浏览场景。

---

## 十六、品牌与风格规范

- 如果用户提供品牌色、品牌字体、品牌元素，应优先遵守品牌体系。
- 如果没有品牌约束，默认采用主流商业 H5 的清晰、简洁、现代化风格。
- 页面风格应统一，不要同页混入多种设计语言。
- "高级感""科技感""年轻化""节日感"等风格表达，必须建立在清晰信息结构之上。

## 十八、输出与交付规范

- 设计输出必须具备明确模块边界和命名。
- 页面中重复出现的视觉结构应按组件思路复用设计。
- 对图标、插图、背景图、商品图、头像图等素材应有清晰分类。
- 文案需区分标题、正文、注释、按钮、提示语等类型。
- 若页面包含弹窗、切换态、选中态、空状态等，设计稿中应补齐对应状态。

## 十九、开发约定

1. **设计稿单位换算**：所有数值基于 750px 设计稿。**CSS/Tailwind 中直接写设计稿 px 值，不手动除以 2**，`postcss-mobile-forever`（`viewportWidth: 750`）构建时自动转换为 vw，在不同屏幕上等比缩放。JS 运行时尺寸不会经过 PostCSS，必须使用 `@new-type/utils` 的 `vw()` 工具函数。
2. **背景图优先**：绝大多数装饰性背景（区块边框、按钮、卡片）均为图片资源（PNG/WebP），**不使用纯 CSS 实现**。
3. **字体加载**：需引入 Roboto（Black/Bold/Medium 字重）和 Shrikhand（排行榜标题），优先使用 CDN 加载。
4. **图层顺序**：背景图片用 `position: absolute; inset: 0` 铺满，内容层用 `position: relative; z-index` 叠加。
5. **倒计时**：显示格式为 `00 Day 00 h 00 m 00 s`，字体 Roboto Bold。
6. **进度条**：由外框（bg-track）+ 内填充（fill，动态宽度）组成，宽度根据数据比例计算。

### 21.3 通用视觉样式规范

#### 21.3.A 标题文字 — 双层叠加描边技法

所有物料中主标题（`#Title.1`）采用**两层文字叠加**渲染，模拟「带描边的金色渐变」效果。

| 层级 | 描边 | 填充 | 说明 |
|---|---|---|---|
| 第一层（底层） | `stroke_width=6`，黑色 | 黑色 | 形成 6px 居中黑色外轮廓 |
| 第二层（上层） | 无描边 | 金色渐变（top→bottom） | 覆盖底层中心区域 |

金色渐变色值：`(255,255,190)` → `(255,210,0)` → `(180,100,5)`

#### 21.3.B 副标题文字 — 双层叠加描边技法

所有物料中副标题（`#Sub-Title.1`）字重固定为 **Bold**，双层叠加。

| 层级 | 描边 | 填充 | 说明 |
|---|---|---|---|
| 第一层（底层） | `stroke_width=4`，白色 | 白色 | 4px 居中白色外轮廓 |
| 第二层（上层） | 无描边 | 黑色 `(0,0,0,255)` | 覆盖底层中心区域 |

**字重约束：** 所有语言副标题统一使用 **Bold** 字重字体。

#### 21.3.C 按钮（CTA Button）— 统一视觉规范

| 属性 | 值 |
|---|---|
| **填充渐变**（竖向） | `(249,255,184)` 0% → `(251,255,101)` 45% → `(255,175,21)` 100% |
| **描边渐变**（OUTSIDE，3px） | `(255,255,255)` 0% → `(255,161,22)` 48% → `(200,96,0)` 100% |
| **圆角** | **60px**（固定值） |
| **内阴影** | offset=(0,4), blur=4, 白色 |
| **投影** | offset=(0,4), blur=0, `(203,110,0)` 深琥珀 |
| 文字颜色 | `(60,5,0,255)` 深棕红 |
| 文字字重 | Bold，水平垂直居中 |

#### 21.3.D Banner 日期徽章 — 固定样式

| 属性 | 值 |
|---|---|
| 尺寸 | **204 × 40 px** |
| 位置 | 左下角，`y = banner高度 - 40` |
| 圆角 | **仅右上角 40px**（左上/左下/右下 = 0px） |
| 背景 | 黑色半透明 `(0,0,0,160)` |
| 文字 | Roboto Bold 20px，水平居中，白色，不随语言切换 |

实现方式：全圆角矩形，再把左上/左下/右下三角补回方角（`corner = min(r, bh//2)`）。

Banner 右侧文字布局规则：
- **Sub-Title y 固定：距 banner 底部 130px**：`sub_y = banner_height - 130`
- **Title y 向上动态推算**：`title_y = sub_y - title_h - 26px`（title_h 由实际行高 auto 决定）
- title 与 sub_title 行高均为 **auto**（由字体渲染自然高度决定，不固定行高）
- Title → Sub-Title 间距：固定 **26px**（所有语言一致）
- **Button y 固定：距 banner 底部 28px**：`btn_y = banner_height - 28 - btn_height`

#### 21.3.E 分享图（Sharing）— 自动主体色遮罩

背景图上方统一叠加半透明遮罩，颜色根据背景图主体色自动生成：
- 背景图缩小至 50×50px，取像素 HSV 均值（忽略高亮区域）
- 降低饱和度 30%、亮度 30%，alpha 固定为 **140**
